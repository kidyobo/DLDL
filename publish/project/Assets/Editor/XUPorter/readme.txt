1、projmods是Json格式的配置文件，用来设置需要被放进Xcode工程的资源
	（1）Group：所有由该projmods添加的文件和文件夹所属的Xcode中的group名称
	（2）libs：在Xcode Build Phases中需要添加的动态链接库的名称，比如libz.dylib
	（3）frameworks：在Xcode Build Phases中需要添加的框架的名称，比如Security.framework
	（4）headerpaths：Xcode中编译设置中的Header Search Paths路径
	（5）files：加入工程的文件名，一般用于第三方framework
	（6）folders：加入工程的文件夹，其中所有的文件和文件夹都将被加入工程中，一般SDK包需要被放入
	（7）linker_flags：添加到工程linker flag中的链接配置，比如-ObjC,-licucore (即编译标记)
	（8）excludes：忽略的文件的正则表达式，匹配的文件将不会被加入工程中,一般用于忽略meta文件
------------------------------------------------------------------------------------------------------------------
2、Mods 文件夹用来存放需要被放进Xcode工程里的SDK包
------------------------------------------------------------------------------------------------------------------
3、OCCode文件夹用来存放需要修改的OC代码
	（1）IEditOCCode是OC代码修改类的接口
	（2）OC代码修改类是用来修改OC代码的，主要用来修改UnityAppController的OC代码
	（3）可以定义多个OC修改类，必须在UnityEditor.XCodeEditor.Edit.OC命名空间中，并且实现IEditorOCCode接口。可通过editoc.txt确定最终使用哪个修改类
	（4）editoc.txt中存放选中的OC代码修改类的类名
------------------------------------------------------------------------------------------------------------------
4、Plists文件夹用来存放需要修改的info.plist
	（1）IEditPlist是plist修改类的接口
	（2）plist修改类是用来修改info.plist的
	（3）可以定义多个plist修改类，必须在UnityEditor.XCodeEditor.Edit.Plist命名空间中，并且实现IEditPlist接口。可通过editplist.txt确定最终使用哪个修改类
	（4）editplist.txt中存放选中的plist修改类的类名
------------------------------------------------------------------------------------------------------------------
注意：上面OC修改类和plist修改类的做法是为了满足在不同的渠道切换时需要修改不同的OC代码和plist，请根据实际情况修改代码，使出包更加方便
